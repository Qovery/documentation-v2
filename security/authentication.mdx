## Authentication Methods

### Email & Password

Default authentication method:

- Password requirements enforced
- Password reset functionality
- Email verification required

### Single Sign-On (SSO)

Enterprise feature supporting:

- **SAML 2.0**: Okta, Auth0, Azure AD
- **OAuth 2.0**: Google, GitHub, GitLab
- **OpenID Connect**: Custom providers

#### Configuring SSO

1. Navigate to Organization Settings → SSO
2. Select SSO provider
3. Configure provider details
4. Test connection
5. Enable for organization

### Multi-Factor Authentication (MFA)

Add extra security layer:

- **TOTP**: Time-based one-time passwords (Google Authenticator, Authy)
- **SMS**: Text message verification
- **Email**: Email-based verification

#### Enabling MFA

```bash
# Via Console
1. User Settings → Security
2. Enable MFA
3. Scan QR code with authenticator app
4. Enter verification code
```

## API Authentication

### API Tokens

Generate API tokens for programmatic access:

```bash
# Create token via CLI
qovery token create --name "CI/CD Token" --scope deploy

# Use token
export QOVERY_API_TOKEN="your-token"
```

### Token Scopes

- **Admin**: Full access
- **Deploy**: Deployment operations only
- **Read**: Read-only access
- **Write**: Create and update resources

### Token Expiration

Set expiration for security:

- 30 days
- 90 days
- 1 year
- Never (not recommended)

## Service Accounts

For automated systems:

```hcl
resource "qovery_service_account" "ci_cd" {
  name  = "CI/CD Pipeline"
  scope = "DEPLOY"
  
  permissions = {
    deploy_applications = true
    manage_environments = false
  }
}
```

## Session Management

- **Session Timeout**: 24 hours (configurable)
- **Remember Me**: 30-day persistent session
- **Concurrent Sessions**: Multiple device support

## Audit Logging

Track all authentication events:

- Login attempts (successful and failed)
- Token creation and deletion
- Password changes
- MFA enrollment
