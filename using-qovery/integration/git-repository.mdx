## Overview

Connect your Git repositories to enable automatic deployments whenever you push code. Qovery integrates with all major Git providers and supports mono-repos, branches, tags, and pull request environments.

## Quick Start

<Steps>
  <Step title="Connect Git Provider">
    Authorize Qovery to access your Git provider during onboarding or in Organization Settings
  </Step>

  <Step title="Select Repository">
    When creating an application, select your Git repository from the list
  </Step>

  <Step title="Choose Branch">
    Specify which branch to deploy (e.g., `main`, `develop`, `staging`)
  </Step>

  <Step title="Enable Auto-Deploy">
    Turn on auto-deploy to automatically deploy on every commit
  </Step>
</Steps>

## Features

<AccordionGroup>
  <Accordion title="Automatic Deployments" icon="arrows-rotate">
    Enable auto-deploy to automatically deploy whenever you push to your selected branch
  </Accordion>

  <Accordion title="Branch and Tag Support" icon="code-branch">
    Deploy from specific branches, tags, or even commit SHAs for precise version control
  </Accordion>

  <Accordion title="Preview Environments" icon="eye">
    Automatically create ephemeral environments for pull requests and merge requests
  </Accordion>

  <Accordion title="Mono-repo Support" icon="folder-tree">
    Deploy multiple applications from a single repository using root paths
  </Accordion>

  <Accordion title="Submodules" icon="diagram-nested">
    Automatic support for Git submodules in your repositories
  </Accordion>
</AccordionGroup>
