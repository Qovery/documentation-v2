Deploy a local Kubernetes cluster on your laptop using Docker Desktop. No cloud account or credit card required!

## Prerequisites

<Steps>
  <Step title="Install Docker Desktop">
    Download and install Docker Desktop: [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop)

    **System Requirements:**
    - 4 CPU cores
    - 8 GB RAM minimum
    - macOS, Windows (WSL2), or Linux

    Make sure Docker Desktop is running before proceeding.
  </Step>

  <Step title="Install Qovery CLI">
    Run this command to install the Qovery CLI:

    ```bash
    curl -s https://get.qovery.com | bash
    ```

    <Tip>
    The CLI works on Linux, macOS, and Windows (WSL only).
    </Tip>
  </Step>
</Steps>

## Authenticate with Qovery

<Steps>
  <Step title="Sign Up">
    Create your free account: [console.qovery.com/signup](https://console.qovery.com/signup)
  </Step>

  <Step title="Authenticate CLI">
    Run the auth command:

    ```bash
    qovery auth
    ```

    This opens your browser to complete authentication.

    <Info>
    **Running in a terminal without browser?** Use `qovery auth --headless` instead.
    </Info>
  </Step>

  <Step title="Set Context">
    Configure your CLI context:

    ```bash
    qovery context set
    ```

    Select your organization when prompted.
  </Step>
</Steps>

## Deploy Local Demo Cluster

<Steps>
  <Step title="Start Demo Cluster">
    Deploy a local k3s Kubernetes cluster with Qovery:

    ```bash
    qovery demo up
    ```

    This command:
    - Installs k3s (lightweight Kubernetes)
    - Installs Qovery agent and components
    - Configures networking and ingress
    - Typically takes 5-10 minutes

    <Warning>
    **Demo clusters are for testing only** - not for production use!
    </Warning>
  </Step>

  <Step title="Verify Installation">
    Once complete, visit [console.qovery.com](https://console.qovery.com)

    You should see your demo cluster listed and connected.
  </Step>
</Steps>

## Choose Your Deployment Method

Qovery supports multiple ways to deploy your applications. Choose the method that fits your workflow:

<Tabs>
  <Tab title="Build from Git (Recommended)">
    **Best for:** Building and deploying source code directly

    <Steps>
      <Step title="Navigate to Git Settings">
        In Qovery Console: **Organization Settings** ‚Üí **Git Repository Access**
      </Step>

      <Step title="Connect Your Git Provider">
        <Tabs>
          <Tab title="GitHub">
            1. Click **Connect GitHub**
            2. Authorize Qovery (read access only)
            3. Select repositories to access
          </Tab>
          <Tab title="GitLab">
            1. Click **Connect GitLab**
            2. Authorize Qovery
            3. Select repositories
          </Tab>
          <Tab title="Bitbucket">
            1. Click **Connect Bitbucket**
            2. Authorize Qovery
            3. Select repositories
          </Tab>
        </Tabs>

        <Info>
        Don't have a repo ready? Use one of our samples:
        - [Node.js App](https://github.com/Qovery/simple-example-node)
        - [Python FastAPI](https://github.com/Qovery/simple-example-python-fastapi)
        - [Go App](https://github.com/Qovery/simple-example-go-gin)

        Fork to your account, then connect!
        </Info>
      </Step>
    </Steps>
  </Tab>

  <Tab title="Deploy Pre-Built Containers">
    **Best for:** Deploying existing Docker images from registries

    <Steps>
      <Step title="Connect Container Registry">
        In Qovery Console: **Organization Settings** ‚Üí **Container Registry**

        Choose your registry:
        - **Docker Hub**
        - **AWS ECR**
        - **Google Artifact Registry**
        - **Azure ACR**
        - **GitHub Container Registry**
        - **GitLab Container Registry**
        - **Scaleway Container Registry**
        - **Generic Registry** (Harbor, Artifactory, etc.)
      </Step>

      <Step title="Provide Credentials">
        Enter registry credentials (username/password or access token)

        [Detailed registry setup guides ‚Üí](/integrations/container-registries/overview)
      </Step>
    </Steps>

    <Info>
    You'll select the container image when creating your application in the next step.
    </Info>
  </Tab>

  <Tab title="Deploy Helm Charts">
    **Best for:** Deploying pre-packaged Helm applications

    <Steps>
      <Step title="Connect Helm Repository">
        In Qovery Console: **Organization Settings** ‚Üí **Helm Repository**

        Add your Helm repository URL (e.g., Bitnami, custom chart museum)
      </Step>

      <Step title="Provide Authentication (if needed)">
        For private repositories, provide credentials

        [Detailed Helm setup ‚Üí](/integrations/helm/overview)
      </Step>
    </Steps>

    <Info>
    You'll select the Helm chart when creating a Helm service in the next step.
    </Info>
  </Tab>
</Tabs>

## Deploy Your First App

<Steps>
  <Step title="Create a Project">
    Projects organize your apps.

    1. Click **Projects** in sidebar
    2. Click **Create Project**
    3. Name: "My First Project"
    4. Select cluster: "docker-desktop"
    5. Click **Create**
  </Step>

  <Step title="Create an Environment">
    Environments are deployment stages (dev/staging/prod).

    1. Inside your project, click **Create Environment**
    2. Name: "development"
    3. Type: **DEVELOPMENT**
    4. Click **Create**
  </Step>

  <Step title="Add Your Service">
    The configuration depends on your deployment method:

    <Tabs>
      <Tab title="From Git Repository">
        1. Click **Add Service** ‚Üí **Application**
        2. **General:**
           - Name: (auto-filled from repo name)
        3. **Source:**
           - Source: **Git Repository**
           - Git Provider: (your connected provider)
           - Repository: (select your app)
           - Branch: `main` or `master`
        4. **Build:**
           - Qovery auto-detects your Dockerfile
           - Leave defaults unless you have custom needs
        5. **Port:**
           - Application port: `3000` (or whatever your app uses)
           - Publicly accessible: **Enable**
        6. Click **Create**
      </Tab>

      <Tab title="From Container Registry">
        1. Click **Add Service** ‚Üí **Application**
        2. **General:**
           - Name: "my-app"
        3. **Source:**
           - Source: **Container Registry**
           - Registry: (your connected registry)
           - Image name: `my-org/my-app`
           - Tag: `latest` (or specific version)
        4. **Port:**
           - Container port: `8080` (or whatever your container exposes)
           - Publicly accessible: **Enable**
        5. Click **Create**

        <Info>
        Qovery will pull the image directly from your registry‚Äîno build needed!
        </Info>
      </Tab>

      <Tab title="From Helm Chart">
        1. Click **Add Service** ‚Üí **Helm**
        2. **General:**
           - Name: "my-helm-app"
        3. **Source:**
           - Source: **Helm Repository**
           - Repository: (your connected Helm repo)
           - Chart name: `my-chart`
           - Chart version: (select version)
        4. **Values (optional):**
           - Override default Helm values if needed
           - Example: `replicaCount: 2`
        5. Click **Create**

        <Info>
        Qovery will deploy the Helm chart with your custom values.
        </Info>
      </Tab>
    </Tabs>
  </Step>

  <Step title="Deploy!">
    Click the **Deploy** button (top right)

    What happens next depends on your deployment method:

    <Tabs>
      <Tab title="Git Repository">

## Troubleshooting

<AccordionGroup>
  <Accordion title="Build failed?">
    - Check build logs for missing dependencies
    - Verify Dockerfile syntax
    - Ensure all dependencies are in `package.json`/`requirements.txt`/etc.
  </Accordion>

  <Accordion title="App crashes after deploy?">
    - Check application logs
    - Verify port matches your app (3000, 8080, etc.)
    - Make sure app listens on `0.0.0.0` not `127.0.0.1`
    - Check for missing environment variables
  </Accordion>

  <Accordion title="Can't access URL?">
    - Wait 1-2 minutes for DNS propagation
    - Verify "Publicly accessible" is enabled
    - Check application status is "Running"
  </Accordion>
</AccordionGroup>

## Clean Up Demo

When you're finished testing, destroy the demo cluster:

```bash
qovery demo destroy
```

This removes the local k3s cluster and all Qovery components.
