## Path 1: Deploy Now with Docker Desktop

**Time:** ~15 minutes total | **Cost:** Free

Perfect for learning Qovery without cloud accounts or costs.

### Step 1: Prerequisites

<Steps>
  <Step title="Install Docker Desktop">
    Download and install Docker Desktop: [docker.com/products/docker-desktop](https://www.docker.com/products/docker-desktop)

    **System Requirements:**
    - 4 CPU cores
    - 8 GB RAM minimum
    - macOS, Windows (WSL2), or Linux

    Make sure Docker Desktop is running before proceeding.
  </Step>

  <Step title="Install Qovery CLI">
    Run this command to install the Qovery CLI:

    ```bash
    curl -s https://get.qovery.com | bash
    ```

    <Tip>
    The CLI works on Linux, macOS, and Windows (WSL only).
    </Tip>
  </Step>
</Steps>

### Step 2: Authenticate with Qovery

<Steps>
  <Step title="Sign Up">
    Create your free account: [console.qovery.com/signup](https://console.qovery.com/signup)
  </Step>

  <Step title="Authenticate CLI">
    Run the auth command:

    ```bash
    qovery auth
    ```

    This opens your browser to complete authentication.

    <Info>
    **Running in a terminal without browser?** Use `qovery auth --headless` instead.
    </Info>
  </Step>

  <Step title="Set Context">
    Configure your CLI context:

    ```bash
    qovery context set
    ```

    Select your organization when prompted.
  </Step>
</Steps>

### Step 3: Deploy Local Demo Cluster

<Steps>
  <Step title="Start Demo Cluster">
    Deploy a local k3s Kubernetes cluster with Qovery:

    ```bash
    qovery demo up
    ```

    This command:
    - Installs k3s (lightweight Kubernetes)
    - Installs Qovery agent and components
    - Configures networking and ingress
    - Typically takes 5-10 minutes

    <Warning>
    **Demo clusters are for testing only** - not for production use!
    </Warning>
  </Step>

  <Step title="Verify Installation">
    Once complete, visit [console.qovery.com](https://console.qovery.com)

    You should see your demo cluster listed and connected.
  </Step>
</Steps>

### Step 5: Deploy Your First App

<Steps>
  <Step title="Create a Project">
    Projects organize your apps.

    1. Click **Projects** in sidebar
    2. Click **Create Project**
    3. Name: "My First Project"
    4. Select cluster: "docker-desktop"
    5. Click **Create**
  </Step>

  <Step title="Create an Environment">
    Environments are deployment stages (dev/staging/prod).

    1. Inside your project, click **Create Environment**
    2. Name: "development"
    3. Type: **DEVELOPMENT**
    4. Click **Create**
  </Step>

  <Step title="Add Your Service">
    The configuration depends on your deployment method from Step 4:

    <Tabs>
      <Tab title="From Git Repository">
        1. Click **Add Service** ‚Üí **Application**
        2. **General:**
           - Name: (auto-filled from repo name)
        3. **Source:**
           - Source: **Git Repository**
           - Git Provider: (your connected provider)
           - Repository: (select your app)
           - Branch: `main` or `master`
        4. **Build:**
           - Qovery auto-detects your Dockerfile
           - Leave defaults unless you have custom needs
        5. **Port:**
           - Application port: `3000` (or whatever your app uses)
           - Publicly accessible: **Enable**
        6. Click **Create**
      </Tab>

      <Tab title="From Container Registry">
        1. Click **Add Service** ‚Üí **Container**
        2. **General:**
           - Name: "my-app"
        3. **Source:**
           - Source: **Container Registry**
           - Registry: (your connected registry)
           - Image name: `my-org/my-app`
           - Tag: `latest` (or specific version)
        4. **Port:**
           - Container port: `8080` (or whatever your container exposes)
           - Publicly accessible: **Enable**
        5. Click **Create**

        <Info>
        Qovery will pull the image directly from your registry‚Äîno build needed!
        </Info>
      </Tab>

      <Tab title="From Helm Chart">
        1. Click **Add Service** ‚Üí **Helm**
        2. **General:**
           - Name: "my-helm-app"
        3. **Source:**
           - Source: **Helm Repository**
           - Repository: (your connected Helm repo)
           - Chart name: `my-chart`
           - Chart version: (select version)
        4. **Values (optional):**
           - Override default Helm values if needed
           - Example: `replicaCount: 2`
        5. Click **Create**

        <Info>
        Qovery will deploy the Helm chart with your custom values.
        </Info>
      </Tab>
    </Tabs>
  </Step>

  <Step title="Deploy!">
    Click the **Deploy** button (top right)

    What happens next depends on your deployment method:

    <Tabs>
      <Tab title="Git Repository">
        Watch Qovery build and deploy:

        - üì• Pulling image from registry

        - ‚úÖ Configuring networking

      </Tab>

      <Tab title="Helm Chart">
        Watch Qovery deploy your Helm chart:
        - üì• Fetching Helm chart
        - ‚öôÔ∏è Rendering templates with values
        - üöÄ Deploying to Kubernetes
        - ‚úÖ Configuring resources

        **First deploy:** 1-3 minutes
      </Tab>
    </Tabs>
  </Step>

  <Step title="Access Your App">
    Once status shows **Running**:
    1. Click your application name
    2. Find the **Public URL**
    3. Click to open in browser

    üéâ Your app is live!
  </Step>
</Steps>

<Accordion title="‚ö†Ô∏è Deployment Failed? Troubleshooting Tips">
  **Build failed?**
  - Check build logs for missing dependencies
  - Verify Dockerfile syntax
  - Ensure all dependencies are in `package.json`/`requirements.txt`/etc.

  **App crashes after deploy?**
  - Check application logs
  - Verify port matches your app (3000, 8080, etc.)
  - Make sure app listens on `0.0.0.0` not `127.0.0.1`
  - Check for missing environment variables

  **Can't access URL?**
  - Wait 1-2 minutes for DNS propagation
  - Verify "Publicly accessible" is enabled
  - Check application status is "Running"
</Accordion>

## Path 2: Production-Ready Setup (AWS/GCP/Azure)

**Time:** ~30 minutes total | **Cost:** Cloud provider charges apply

Choose your cloud provider and follow the steps to create production infrastructure.

<Tabs>
  <Tab title="AWS">
    ### Prerequisites

    Before starting:
    - AWS account

    ### Step 1: Create AWS Credentials

    <Tabs>
      <Tab title="STS Assume Role (Recommended)">
        **Most secure** - Short-lived, automatically refreshed credentials with granular access control.

        <Warning>
        **We strongly recommend using Assume Role via STS.** Static credentials are long-lived and more exposed to leaks, while Assume Role provides short-lived, automatically refreshed credentials with granular access control, reducing security risks.
        </Warning>

        <Steps>
          <Step title="Access AWS Console">
            - Log into [console.aws.amazon.com](https://console.aws.amazon.com)
            - Ensure you're on the correct AWS account
          </Step>

          <Step title="Deploy CloudFormation Stack">
            - In Qovery Console, click the **CloudFormation link** provided
            - This opens AWS CloudFormation with a pre-built template
            - Click **Next** ‚Üí **Next** ‚Üí **Next**
            - ‚úÖ Check **"I acknowledge that AWS CloudFormation might create IAM resources with custom names"**
            - Click **Create stack**
          </Step>

          <Step title="Wait for Stack Creation">
            Wait ~1 minute for stack status: `CREATE_COMPLETE`
          </Step>

          <Step title="Retrieve Role ARN">
            - Go to the **Outputs** tab
            - Copy the **RoleArn** value (starts with `arn:aws:iam::`)

            ![Stack Output](/images/aws-credentials/stack-output.png)
          </Step>

          <Step title="Configure in Qovery">
            - Paste the Role ARN into Qovery
            - Give your credentials a name for easy identification
            - Click **Save**
          </Step>
        </Steps>
      </Tab>

      <Tab title="Static Credentials">
        **Alternative** - Access Key and Secret Access Key (long-lived credentials).

        <Warning>
        Static credentials are more exposed to leaks and require manual rotation. Use STS Assume Role for production environments.
        </Warning>

        <Steps>
          <Step title="Create IAM User">
            - Go to AWS IAM Console ‚Üí **Users**
            - Click **Create user**

            ![Create User](/images/aws-credentials/aws-create-user-1.jpg)
          </Step>

          <Step title="Attach Permissions">
            - User name: `qovery-admin`
            - Attach **AdministratorAccess** policy (or custom policy with required permissions)

            ![Attach Policy](/images/aws-credentials/aws-add-policy-1.jpg)
          </Step>

          <Step title="Generate Access Keys">
            - Go to user's **Security credentials** tab
            - Click **Create access key**
            - Select use case: **Third-party service**
            - Copy **Access Key ID** and **Secret Access Key**

            ![Create Access Key](/images/aws-credentials/aws-create-credentials-1.png)
          </Step>

          <Step title="Configure in Qovery">
            - In Qovery, select **Static Credentials**
            - Paste Access Key ID and Secret Access Key
            - Click **Save**

            <Warning>
            Remember to rotate these credentials regularly (recommended every 90 days).
            </Warning>
          </Step>
        </Steps>
      </Tab>
    </Tabs>

    ### Step 2: Create Cluster

    <Steps>
      <Step title="Navigate to Qovery">

      <Step title="Configure Cluster">

        - **Region:** Choose closest to users (e.g., `us-east-1`, `eu-west-1`)

      <Step title="Select Instance Types">
        **Select ALL instance types** Karpenter can choose from:

        Example selection:
        - ‚úÖ **t3.medium** (2 vCPU, 4GB)
        - ‚úÖ **t3.large** (2 vCPU, 8GB)
        - ‚úÖ **t3.xlarge** (4 vCPU, 16GB)
        - ‚úÖ **t3.2xlarge** (8 vCPU, 32GB)
        - ‚úÖ **m5.large** (2 vCPU, 8GB)
        - ‚úÖ **m5.xlarge** (4 vCPU, 16GB)

        </Info>

      <Step title="Create & Deploy">
        Click **Create and Deploy**
