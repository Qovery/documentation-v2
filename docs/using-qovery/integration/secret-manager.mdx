## Overview

Integrate Qovery with external secret managers to centralize secret management across your organization. Secrets are automatically synchronized and injected as environment variables in your applications.

## Integration Methods

<AccordionGroup>
  <Accordion title="Doppler Integration" icon="d">
    **Native Qovery Integration**

    Connect Doppler directly in Qovery Console:
    - Organization or project-level integration
    - Automatic secret synchronization
    - Map Doppler projects to Qovery environments
    - Secrets updated in real-time

    [Setup Guide →](/integrations/secret-managers/doppler)
  </Accordion>

  <Accordion title="AWS Secrets Manager" icon="aws">
    **Cloud-Native Secret Storage**

    Use AWS Secrets Manager for AWS deployments:
    - IAM-based access control
    - Automatic rotation support
    - Audit logging via CloudTrail
    - Regional secret replication

    [Setup Guide →](/integrations/secret-managers/aws-secrets-manager)
  </Accordion>

  <Accordion title="Qovery Environment Variables" icon="lock">
    **Built-in Encrypted Storage**

    Use Qovery's native secret management:
    - Encrypted at rest
    - RBAC-controlled access
    - No external dependencies
    - Variable aliases and overrides

    [Environment Variables →](/configuration/environment-variables)
  </Accordion>
</AccordionGroup>

## Quick Start

### Doppler Integration

<Steps>
  <Step title="Generate Doppler Token">
    Create a service token in your Doppler project
  </Step>

  <Step title="Connect to Qovery">
    Add Doppler integration in Organization or Project Settings
  </Step>

  <Step title="Map Projects">
    Link Doppler projects to Qovery environments
  </Step>

  <Step title="Deploy">
    Secrets automatically sync and inject into applications
  </Step>
</Steps>

### External Secrets Operator

<Steps>
  <Step title="Install Operator">
    Deploy External Secrets Operator to your cluster
  </Step>

  <Step title="Configure Secret Store">
    Create SecretStore pointing to your secret backend
  </Step>

  <Step title="Create External Secret">
    Define which secrets to sync to Kubernetes
  </Step>

  <Step title="Use in Qovery">
    Reference synced secrets in your applications
  </Step>
</Steps>
