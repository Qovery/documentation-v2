## Overview

Qovery works with any Kubernetes cluster, whether managed by cloud providers or self-managed. Choose between **Qovery-managed clusters** (fully automated) or **Bring Your Own Kubernetes (BYOK)** for maximum control.

## Deployment Options

<CardGroup cols={2}>
  <Card title="Qovery-Managed" icon="wand-magic-sparkles">
    **Zero Configuration**

    Qovery provisions and manages your Kubernetes cluster on your cloud provider account. Automated updates, scaling, and maintenance.

    ✅ AWS EKS
    ✅ GCP GKE
    ✅ Azure AKS
    ✅ Scaleway Kapsule
  </Card>

  <Card title="Bring Your Own Kubernetes (BYOK)" icon="server">
    **Full Control**

    Connect your existing Kubernetes cluster to Qovery. You manage the cluster, Qovery manages deployments.

    ✅ Any Kubernetes 1.24+
    ✅ Self-managed clusters
    ✅ On-premise
    ✅ Edge locations
  </Card>
</CardGroup>

## Qovery-Managed vs BYOK

<Tabs>
  <Tab title="Qovery-Managed">
    **What Qovery Manages**:
    - ✅ Cluster provisioning and setup
    - ✅ Kubernetes version upgrades
    - ✅ Node pool management
    - ✅ Auto-scaling configuration
    - ✅ Security patches
    - ✅ Monitoring and logging setup
    - ✅ Network configuration
    - ✅ SSL/TLS certificates

    **Benefits**:
    - Zero cluster management overhead
    - Automated best practices
    - Fast setup (15-30 minutes)
    - Optimized for Qovery workloads
    - Included in Qovery pricing

    **Best For**:
    - Teams without Kubernetes expertise
    - Startups and small teams
    - Rapid deployment needs
    - Standard use cases
  </Tab>

  <Tab title="BYOK (Bring Your Own)">
    **What You Manage**:
    - ✅ Cluster provisioning
    - ✅ Kubernetes version upgrades
    - ✅ Node pool management
    - ✅ Security and compliance
    - ✅ Network policies
    - ✅ Backup and disaster recovery

    **What Qovery Manages**:
    - ✅ Application deployments
    - ✅ Environment management
    - ✅ Secret management
    - ✅ CI/CD pipelines

    **Benefits**:
    - Full cluster control
    - Custom configurations
    - Meet specific compliance requirements
    - Use existing clusters
    - Multi-tenant setups

    **Best For**:
    - Enterprise with existing clusters
    - Specific compliance needs (HIPAA, PCI-DSS)
    - Custom networking requirements
    - Multi-tenancy requirements
  </Tab>
</Tabs>

    - Integrated with GCP services (Cloud SQL, GCS, etc.)

    **Regions**: All GCP regions

  <Accordion title="Azure AKS" icon="microsoft">
    **Qovery-Managed Features**:

    - VNet and subnet configuration

    - Virtual Nodes (ACI integration)

    - Integrated with Azure services (SQL, Storage, etc.)

    **Versions**: 1.24, 1.25, 1.26, 1.27, 1.28, 1.29
    **Regions**: All Azure regions
    **Node Types**: Standard, Spot, Virtual Nodes
  </Accordion>

  <Accordion title="Scaleway Kapsule" icon="/images/logos/cloud-providers/scaleway-icon-light.svg">
    **Qovery-Managed Features**:

    - Private network configuration
    - Auto-scaling with Cluster Autoscaler
    - Cost-effective pricing
    - European data centers
    - GDPR compliance
    - Integrated with Scaleway services (Database, Object Storage)

    **Versions**: 1.24, 1.25, 1.26, 1.27, 1.28

    **Node Types**: DEV1, GP1, PRO2 (various sizes)
  </Accordion>

  <Accordion title="EKS Anywhere" icon="building">
    **BYOK Features**:
    - AWS EKS on your infrastructure
    - Consistent with EKS API
    - AWS support and tooling
    - On-premise or edge deployment
    - VMware vSphere support
    - Bare metal support

    **Requirements**:
    - Kubernetes 1.24+
    - kubectl access
    - Qovery agent installation
    - Network connectivity to Qovery
  </Accordion>

  <Accordion title="Generic Kubernetes (BYOK)" icon="dharmachakra">
    **Supported Distributions**:
    - Vanilla Kubernetes
    - Rancher
    - OpenShift
    - k3s
    - Tanzu
    - Any certified Kubernetes

    **Requirements**:
    - Kubernetes 1.24+
    - kubectl admin access
    - LoadBalancer support (MetalLB for on-premise)
    - Ingress controller
    - Storage class (for persistent volumes)
    - Network connectivity to Qovery Control Plane
  </Accordion>

  <Accordion title="Docker (Local)" icon="docker">
    **Local Development Options**:
    - Docker Desktop (Mac/Windows): Built-in Kubernetes
    - Minikube: Lightweight Kubernetes
    - Kind: Kubernetes in Docker
    - k3d: Lightweight k3s in Docker

    **Use Cases**:
    - Local development and testing
    - CI/CD pipelines (ephemeral environments)
    - Learning Kubernetes
    - Pre-production validation

    **Limitations**:
    - Not for production workloads
    - Limited resources
    - Single-node (typically)
    - No high availability
  </Accordion>
</AccordionGroup>
