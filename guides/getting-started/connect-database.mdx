## Creating a Database

<Steps>
  <Step title="Navigate to Environment">
    Go to your [environment](/configuration/environment) in the Qovery Console
  </Step>

  <Step title="Add Database">
    Click **Add Database**
  </Step>

  <Step title="Configure Database">
    Select and configure your database:

    **Type**: PostgreSQL, MySQL, MongoDB, or Redis
    **Version**: Choose the version
    **Mode**: 
    - **Managed**: Cloud provider service (RDS, Cloud SQL)
    - **Container**: Database in your cluster

    **Resources**:
    ```yaml
    CPU: 500m
    Memory: 1024 MB
    Storage: 20 GB
    ```
  </Step>

  <Step title="Create">
    Click **Create Database**
  </Step>
</Steps>

## Connecting Your Application

Qovery automatically injects connection information as [environment variables](/configuration/environment-variables).

### PostgreSQL Connection

```javascript
// Node.js example
const { Pool } = require('pg');

const pool = new Pool({
  host: process.env.DATABASE_HOST,
  port: process.env.DATABASE_PORT,
  database: process.env.DATABASE_NAME,
  user: process.env.DATABASE_USER,
  password: process.env.DATABASE_PASSWORD,
});
```

### Environment Variables

Qovery provides these variables automatically:

```bash
DATABASE_HOST=postgres-xyz.internal
DATABASE_PORT=5432
DATABASE_NAME=mydatabase
DATABASE_USER=admin
DATABASE_PASSWORD=<auto-generated-secret>
```

## Database Management

### Backups

Qovery automatically backs up your databases:

- **Frequency**: Daily
- **Retention**: 30 days (configurable)
- **Point-in-Time Recovery**: Available for managed databases

### Restore from Backup

<Steps>
  <Step title="Access Database Settings">
    Click on your database â†’ **Backups**
  </Step>

  <Step title="Select Backup">
    Choose the backup to restore from
  </Step>

  <Step title="Restore">
    Click **Restore** and confirm
  </Step>
</Steps>

## Best Practices

<Tip>
Use **Managed databases** for production workloads for better reliability and automatic backups.
</Tip>

<Tip>
Use **Container databases** for development and testing to save costs.
</Tip>

<Warning>
Database passwords are generated automatically. Store them securely and rotate regularly.
</Warning>
