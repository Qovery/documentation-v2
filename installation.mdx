## What Happens During Installation?

### Qovery-Managed Installation

When you choose Qovery-Managed installation (AWS, GCP, Azure, Scaleway), Qovery automatically provisions:

<Tabs>
  <Tab title="Infrastructure">
    **Cloud Resources**:
    - Kubernetes cluster with production configuration
    - VPC/Network with public and private subnets
    - Load balancers and ingress controllers
    - NAT Gateways for private subnet internet access
    - IAM roles and security groups
  </Tab>

  <Tab title="Kubernetes Components">
    **Installed Services**:
    - Auto-scaler (Karpenter or cluster-autoscaler)
    - AWS Load Balancer Controller (or equivalent)
    - Ingress Nginx for routing
    - Cert-manager for SSL certificates
    - CoreDNS for service discovery
    - Metrics Server for resource monitoring
  </Tab>

  <Tab title="Qovery Agent">
    **Qovery Components**:
    - Qovery Engine (deployment orchestration)
    - Qovery Shell Agent (remote access)
    - Logging agents (Loki, Promtail)
    - Metrics collection (Prometheus)
    - Event streaming
  </Tab>

  <Tab title="Time & Cost">
    **Provisioning Time**:
    - AWS EKS: 20-30 minutes
    - GCP GKE: 15-20 minutes
    - Azure AKS: 20-25 minutes
    - Scaleway Kapsule: 15-20 minutes

    **Monthly Cost Estimate**:
    - Control Plane: ~$70-150/month (varies by provider)
    - Worker Nodes: Based on instance types selected
    - Network: ~$100-200/month (NAT, Load Balancers)

    Use spot/preemptible instances for significant savings.
  </Tab>
</Tabs>

### BYOK Installation

For BYOK (Bring Your Own Kubernetes), you install the Qovery Agent via Helm:

```bash
# Add Qovery Helm repository
helm repo add qovery https://helm.qovery.com
helm repo update

# Install Qovery Agent
helm install qovery-agent qovery/qovery-agent \
  --namespace qovery \
  --create-namespace \
  --values qovery-values.yaml
```

**What Gets Installed**:
- Qovery Agent (deployment orchestration)
- Logging and metrics collectors
- Shell agent (optional, for remote access)
- Ingress configuration (if not present)

**Your cluster remains**:
- Under your full control
- With your existing configurations
- Using your preferred tools and workflows

[BYOK Installation Guide →](/integrations/kubernetes/byok)

## Minimum Requirements

<Warning>
Ensure your cluster meets these minimum requirements before installation
</Warning>

| Resource | Minimum | Recommended | Notes |
|----------|---------|-------------|-------|
| **Kubernetes Version** | 1.24+ | 1.28+ | Latest stable version recommended |
| **Worker Nodes** | 2 | 3+ | For high availability |
| **CPU per Node** | 2 cores | 4+ cores | More for production workloads |
| **Memory per Node** | 4 GB | 8+ GB | Depends on application requirements |
| **Disk per Node** | 20 GB | 50+ GB | For container images and logs |
| **Network** | Basic | VPC/Private | Private subnets recommended |

**For Production**:
- Use at least 3 worker nodes across multiple availability zones
- Enable auto-scaling
- Use private subnets with NAT gateway
- Enable logging and monitoring
- Regular automated backups
