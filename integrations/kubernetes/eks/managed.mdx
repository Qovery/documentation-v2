## Overview

Qovery-managed EKS provides production-ready Kubernetes clusters on AWS with zero configuration. Qovery handles everything: cluster creation, networking, scaling, monitoring, and ongoing maintenance.

**Perfect for**: Teams who want AWS but don't want to manage Kubernetes infrastructure.

## Create Your First Cluster

<Steps>
  <Step title="Open Qovery Console">
    1. Go to **Organization Settings** → **Clusters**
    2. Click **Create Cluster**
    3. Select **AWS**
  </Step>

  <Step title="Configure Cluster">
    - **Name**: e.g., `production-eks`
    - **Region**: Choose closest to your users (e.g., `us-east-1`)
  </Step>

  <Step title="Connect AWS Account">
    Choose how to connect your AWS account:

    <Tabs>
      <Tab title="STS Assume Role (Recommended)">
        **Most secure** - Temporary credentials that auto-rotate.

        1. Click the **CloudFormation link** in Qovery
        2. In AWS, click **Next** → **Next** → check IAM acknowledgement → **Create stack**
        3. Wait 1 minute for completion
        4. Copy the **RoleArn** from Outputs tab
        5. Paste back in Qovery and click **Save**
      </Tab>

      <Tab title="Static Credentials">
        **Alternative** - AWS Access Keys (manual rotation required).

        1. In AWS IAM, create user with **AdministratorAccess**
        2. Generate **Access Key ID** and **Secret Access Key**
        3. In Qovery, select **Static Credentials**
        4. Enter your keys and click **Save**

        <Warning>Rotate keys every 90 days. STS Assume Role recommended for production.</Warning>
      </Tab>
    </Tabs>
  </Step>

  <Step title="Choose Instance Size">
    - **t3.medium** - Testing
    - **t3.large** - Light production (recommended for starters)
    - **t3.xlarge** - Standard production
  </Step>

  <Step title="Create">
    Click **Create and Deploy** - your cluster will be ready in 20-30 minutes!
  </Step>
</Steps>

## What Qovery Creates

- ✅ **EKS Cluster** - Latest stable Kubernetes
- ✅ **VPC & Networking** - Public/private subnets across 3 AZs
- ✅ **NAT Gateways** - Secure internet access
- ✅ **Security Groups & IAM Roles** - Pre-configured best practices
- ✅ **Karpenter** - Intelligent auto-scaling (save up to 60% on costs)
- ✅ **AWS Load Balancer Controller** - Automatic ingress management
- ✅ **EBS CSI Driver** - Persistent volume support
- ✅ **Metrics Server** - Resource monitoring
- ✅ **Qovery Agent** - Observability and management

### Karpenter Auto-Scaling

Qovery uses [Karpenter](https://karpenter.sh/) to automatically provision optimal EC2 instances:

- Scales nodes within seconds based on workload demands
- Consolidates workloads to reduce costs
- Handles spot instance interruptions gracefully
- Supports wide range of instance types (t3, m5, m6i, c5, r5, GPU)
- Mix of on-demand and spot instances for reliability

<Tip>
Configure Karpenter node pools and instance types in **Cluster Settings** after creation. Learn more in [Cluster Configuration](/configuration/clusters).
</Tip>

## Configuration Options

### Cluster Settings

**Instance Types**:
- General Purpose: t3, m5, m6i, m6g (Graviton)
- Compute Optimized: c5, c6i, c6g (Graviton)
- Memory Optimized: r5, r6i, r6g (Graviton)

**Auto-Scaling**:
- Min nodes: 1
- Max nodes: 100
- Karpenter automatically provisions optimal instances

**Spot Instances**:
- Enable Spot instances for cost savings (60-90% off)
- Qovery handles interruptions gracefully
- Mix of Spot and On-Demand for reliability

### Network Settings

**VPC Configuration**:
- CIDR: /16 (65,536 IPs)
- 3 public subnets (load balancers)
- 3 private subnets (pods)
- NAT Gateways per AZ

**Access**:
- Public endpoint (default)
- Private endpoint (enterprise feature)
- VPN access for private endpoints
