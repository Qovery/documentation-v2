## Overview

Qovery-managed EKS provides production-ready Kubernetes clusters on AWS with zero configuration. Qovery handles everything: cluster creation, networking, scaling, monitoring, and ongoing maintenance.

**Perfect for**: Teams who want AWS but don't want to manage Kubernetes infrastructure.

### Karpenter-Powered Auto-Scaling

All Qovery-managed EKS clusters use [Karpenter](https://karpenter.sh/) for intelligent node provisioning:

- **60% cost savings** through spot instances and consolidation
- **Fast scaling** in seconds (not minutes)
- **Smart instance selection** across multiple instance families
- **Automatic workload optimization** to minimize costs

<Info>
When you select an instance type (like t3.large), Karpenter uses it as a baseline and can provision similar instances (t3, m5, m6i families) to optimize cost and availability.
</Info>

## Create Your First Cluster

<Steps>
  <Step title="Open Qovery Console">
    1. Go to **Organization Settings** → **Clusters**
    2. Click **Create Cluster**
    3. Select **AWS**
  </Step>

  <Step title="Configure Cluster">
    - **Name**: e.g., `production-eks`
    - **Region**: Choose closest to your users (e.g., `us-east-1`)
  </Step>

  <Step title="Connect AWS Account">
    Choose how to connect your AWS account:

    <Tabs>
      <Tab title="STS Assume Role (Recommended)">
        **Most secure** - Temporary credentials that auto-rotate automatically.

        1. Click the **CloudFormation link** in Qovery
        2. In AWS CloudFormation Console:
           - Click **Next** → **Next** → **Next**
           - ✅ Check **"I acknowledge that AWS CloudFormation might create IAM resources"**
           - Click **Create stack**
        3. Wait 1 minute for `CREATE_COMPLETE` status
        4. Go to **Outputs** tab
        5. Copy the **RoleArn** value (starts with `arn:aws:iam::`)
        6. Paste back in Qovery and click **Save**

        <Tip>
        CloudFormation creates an IAM role that uses AWS STS for temporary credentials. More secure than static access keys.
        </Tip>
      </Tab>

      <Tab title="Static Credentials">
        **Alternative** - AWS Access Keys (manual rotation required every 90 days).

        1. In AWS IAM Console, go to **Users** → **Create user**
        2. User name: `qovery-admin`
        3. Select **Attach policies directly** → Choose **AdministratorAccess**
        4. Click **Create user**
        5. Open user → **Security credentials** → **Create access key**
        6. Select **Third-party service** → **Create access key**
        7. Copy **Access Key ID** and **Secret Access Key**
        8. In Qovery, select **Static Credentials**
        9. Paste your keys and click **Save**

        <Warning>
        Static credentials require manual rotation every 90 days. STS Assume Role recommended for production environments.
        </Warning>
      </Tab>
    </Tabs>

    For detailed instructions with policy information, see [AWS Installation Guide](/installation/aws#connect-aws-account).
  </Step>

  <Step title="Choose Instance Size">
    Select baseline instance type (Karpenter optimizes from here):

    - **t3.medium** (2 vCPU, 4GB) - Testing ($30/month)
    - **t3.large** (2 vCPU, 8GB) - Light production ($60/month) ← **Recommended**
    - **t3.xlarge** (4 vCPU, 16GB) - Standard production ($120/month)

    <Info>
    Karpenter can provision similar instances (t3, m5, m6i families) based on workload needs. Enable spot instances for 60-90% savings.
    </Info>
  </Step>

  <Step title="Create">
    Click **Create and Deploy** - your cluster will be ready in 20-30 minutes!
  </Step>
</Steps>

### Karpenter Auto-Scaling

Qovery uses [Karpenter](https://karpenter.sh/) to automatically provision optimal EC2 instances:

- Scales nodes within seconds based on workload demands
- Consolidates workloads to reduce costs
- Handles spot instance interruptions gracefully
- Supports wide range of instance types (t3, m5, m6i, c5, r5, GPU)
- Mix of on-demand and spot instances for reliability

<Tip>
Configure Karpenter node pools and instance types in **Cluster Settings** after creation. Learn more in [Cluster Configuration](/configuration/clusters).
</Tip>

## Configuration Options

### Cluster Settings

**Instance Types**:
- General Purpose: t3, m5, m6i, m6g (Graviton)
- Compute Optimized: c5, c6i, c6g (Graviton)
- Memory Optimized: r5, r6i, r6g (Graviton)

**Auto-Scaling**:
- Min nodes: 1
- Max nodes: 100
- Karpenter automatically provisions optimal instances

**Spot Instances**:
- Enable Spot instances for cost savings (60-90% off)
- Qovery handles interruptions gracefully
- Mix of Spot and On-Demand for reliability

### Network Settings

**VPC Configuration**:
- CIDR: /16 (65,536 IPs)
- 3 public subnets (load balancers)
- 3 private subnets (pods)
- NAT Gateways per AZ

**Access**:
- Public endpoint (default)
- Private endpoint (enterprise feature)
- VPN access for private endpoints
