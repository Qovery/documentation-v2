## Overview

Qovery-managed EKS provides zero-configuration Kubernetes clusters on AWS. Qovery creates, configures, and maintains your EKS cluster with best practices built-in.

## Quick Start

<Steps>
  <Step title="Connect AWS Account">
    **Set up AWS credentials** (choose one method):

    - **STS Assume Role** (Recommended): Use CloudFormation to create a secure IAM role
    - **Static Credentials**: Use Access Key ID and Secret Access Key

    See the [detailed AWS installation guide](/installation/aws#step-2-attach-aws-credentials) for step-by-step instructions on both methods.
  </Step>

  <Step title="Create Cluster">
    Settings → Clusters → Create Cluster
    - Name: `production-eks`
    - Provider: AWS
    - Region: `us-east-1`
    - Select your credentials
    - Click **Create**
  </Step>

  <Step title="Wait for Provisioning">
    Cluster creation takes 20-30 minutes:
    - VPC and networking setup
    - EKS control plane creation
    - Worker node provisioning
    - Qovery components installation
  </Step>

  <Step title="Deploy Applications">
    Once cluster is ready, start deploying applications
  </Step>
</Steps>

<Info>
For complete installation instructions including credential setup, see the [AWS Installation Guide](/installation/aws).
</Info>

## What Qovery Creates

- ✅ EKS cluster (latest stable Kubernetes)
- ✅ VPC with public/private subnets across 3 AZs
- ✅ NAT Gateways for internet access
- ✅ Security groups and IAM roles
- ✅ Karpenter for auto-scaling
- ✅ AWS Load Balancer Controller
- ✅ EBS CSI driver
- ✅ Metrics server
- ✅ Qovery agent and observability stack

## Configuration Options

### Cluster Settings

**Instance Types**:
- General Purpose: t3, m5, m6i, m6g (Graviton)
- Compute Optimized: c5, c6i, c6g (Graviton)
- Memory Optimized: r5, r6i, r6g (Graviton)

**Auto-Scaling**:
- Min nodes: 1
- Max nodes: 100
- Karpenter automatically provisions optimal instances

**Spot Instances**:
- Enable Spot instances for cost savings (60-90% off)
- Qovery handles interruptions gracefully
- Mix of Spot and On-Demand for reliability

### Network Settings

**VPC Configuration**:
- CIDR: /16 (65,536 IPs)
- 3 public subnets (load balancers)
- 3 private subnets (pods)
- NAT Gateways per AZ

**Access**:
- Public endpoint (default)
- Private endpoint (enterprise feature)
- VPN access for private endpoints
