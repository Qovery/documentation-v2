## Deployment Models

### Managed by Qovery

Qovery provisions and manages the Kubernetes cluster for you:

**Features**:
- ✅ One-click cluster creation
- ✅ Automatic Kubernetes upgrades
- ✅ Built-in monitoring and logging
- ✅ Automated backups
- ✅ Security patches and updates
- ✅ High availability configuration
- ✅ Cost optimization

**Best For**:
- Teams without Kubernetes expertise
- Fast time to market
- Simplified operations
- Reduced operational overhead

**Supported On**:
- AWS (EKS)
- Google Cloud (GKE)
- Azure (AKS)
- Scaleway (Kapsule)

### Self-Managed (BYOK)

Bring Your Own Kubernetes - you manage the cluster:

**Features**:
- ✅ Full control over cluster configuration
- ✅ Use existing Kubernetes clusters
- ✅ Custom networking setup
- ✅ Compliance requirements
- ✅ Works with any K8s distribution

**Responsibilities**:
- ❗ Kubernetes version upgrades
- ❗ Cluster maintenance
- ❗ Security patches
- ❗ Backup and disaster recovery
- ❗ Monitoring setup

**Best For**:
- Kubernetes experts
- Existing cluster utilization
- Strict compliance requirements
- Custom infrastructure needs

**Supported On**:
- Any CNCF-certified Kubernetes
- EKS, GKE, AKS (self-managed)
- DigitalOcean Kubernetes
- Linode LKE
- On-premise Kubernetes
- K3s, Rancher, OpenShift

## Getting Started

### Choose Your Cloud Provider

Consider these factors when choosing:

<Tabs>
  <Tab title="AWS">
    **Choose AWS if**:
    - Already using AWS services
    - Need widest service selection
    - Require multi-region deployments
    - Want mature, battle-tested infrastructure
    - Need advanced networking (VPC, VPN)

    **Considerations**:
    - More complex pricing
    - Steeper learning curve
    - Best-in-class ecosystem

    **Monthly Cost** (minimum):
    - EKS: $72 (control plane)
    - EC2: ~$150 (3 t3.medium nodes)
    - Load Balancer: ~$20
    - Total: ~$242/month
  </Tab>

  <Tab title="GCP">
    **Choose GCP if**:
    - Need best-in-class Kubernetes (GKE)
    - Want simple, predictable pricing
    - Prefer Google ecosystem
    - Need excellent data analytics tools
    - Want cutting-edge features

    **Considerations**:
    - Smaller service catalog vs AWS
    - Excellent Kubernetes experience
    - Great for data-heavy workloads

    **Monthly Cost** (minimum):
    - GKE: $72 (control plane)
    - Compute: ~$120 (3 e2-medium nodes)
    - Load Balancer: ~$20
    - Total: ~$212/month
  </Tab>

  <Tab title="Azure">
    **Choose Azure if**:
    - Enterprise with Microsoft agreements
    - Need AD/Azure AD integration
    - Want hybrid cloud capabilities
    - Microsoft-centric organization
    - Need strong compliance certifications

    **Considerations**:
    - Best for Microsoft shops
    - Excellent enterprise features
    - Strong hybrid cloud support

    **Monthly Cost** (minimum):
    - AKS: Free (control plane)
    - VMs: ~$140 (3 Standard B2s)
    - Load Balancer: ~$20
    - Total: ~$160/month
  </Tab>

  <Tab title="Scaleway">
    **Choose Scaleway if**:
    - Need European data residency
    - Want GDPR compliance by default
    - Prefer competitive pricing
    - Simple, straightforward services
    - Support European cloud provider

    **Considerations**:
    - EU-focused (data centers in France, Netherlands, Poland)
    - Smaller service catalog
    - Great price/performance ratio

    **Monthly Cost** (minimum):
    - Kapsule: Free (control plane)
    - Instances: ~$80 (3 DEV1-M)
    - Load Balancer: ~$12
    - Total: ~$92/month
  </Tab>
</Tabs>

### Installation Process

The installation process varies by provider:

<Steps>
  <Step title="Prepare Cloud Account">
    - Create cloud account (if needed)
    - Set up billing
    - Configure IAM/service accounts
    - Generate access credentials
  </Step>

  <Step title="Install Qovery CLI">
    ```bash
    curl -s https://get.qovery.com | bash
    ```
  </Step>

  <Step title="Authenticate with Qovery">
    ```bash
    qovery auth
    ```
  </Step>

  <Step title="Install Qovery">
    ```bash
    qovery cluster install
    ```

    Follow the prompts to:
    - Select cloud provider
    - Choose region
    - Configure cluster size
    - Set up networking
  </Step>

  <Step title="Verify Installation">
    - Check cluster status in Qovery Console
    - Deploy test application
    - Verify all services are running
  </Step>
</Steps>

## Multi-Cloud Strategy

### Benefits of Multi-Cloud

- **Avoid Vendor Lock-in**: Switch providers without rewriting applications
- **Disaster Recovery**: Failover to different cloud provider
- **Cost Optimization**: Use best pricing for each workload
- **Compliance**: Meet data residency requirements
- **Performance**: Deploy close to users globally

### Implementing Multi-Cloud

<AccordionGroup>
  <Accordion title="Multiple Clusters">
    Deploy different environments on different clouds:

    ```yaml
    Development: Scaleway (lowest cost)
    Staging: GCP (best Kubernetes)
    Production: AWS (enterprise features)
    DR Site: Azure (backup region)
    ```

    **Pros**: Maximum flexibility, cost optimization
    **Cons**: More complexity, multiple bills
  </Accordion>

  <Accordion title="Hybrid Strategy">
    Mix managed and self-managed:

    ```yaml
    Production: AWS EKS (managed by Qovery)
    On-Premise: Self-managed K8s (BYOK)
    Edge: Multiple regional K8s clusters
    ```

    **Pros**: Leverage existing infrastructure
    **Cons**: Operational complexity
  </Accordion>

  <Accordion title="Progressive Migration">
    Gradually move between providers:

    1. Start on Scaleway (low cost)
    2. Grow to AWS (more services)
    3. Add GCP for specific workloads
    4. Keep Qovery abstraction layer

    **Pros**: Low initial investment, smooth scaling
    **Cons**: Requires planning
  </Accordion>
</AccordionGroup>

## Cloud Provider Services

### Managed Databases

Each cloud provider offers managed database services:

| Database | AWS | GCP | Azure | Scaleway |
|----------|-----|-----|-------|----------|
| PostgreSQL | RDS PostgreSQL | Cloud SQL PostgreSQL | Azure DB PostgreSQL | Managed PostgreSQL |
| MySQL | RDS MySQL | Cloud SQL MySQL | Azure DB MySQL | Managed MySQL |
| MongoDB | DocumentDB | - | Cosmos DB | - |
| Redis | ElastiCache | Memorystore | Azure Cache | - |

**Benefits**:
- Automated backups
- High availability
- Automatic failover
- Point-in-time recovery
- Monitoring included

### Container Registries

Native registry for each cloud:

- **AWS ECR**: Deep EKS integration, IAM-based access
- **GCP Artifact Registry**: Vulnerability scanning, multi-format
- **Azure ACR**: Azure AD integration, geo-replication
- **Scaleway CR**: Simple, cost-effective

### Object Storage

S3-compatible storage on all clouds:

- **AWS S3**: Industry standard, global reach
- **GCP Cloud Storage**: Excellent for analytics
- **Azure Blob Storage**: Hot/cool/archive tiers
- **Scaleway Object Storage**: S3-compatible, European

## Kubernetes Versions

Qovery supports Kubernetes versions across all providers:

### Version Support Policy

- **Supported Versions**: Latest 3 minor versions (e.g., 1.27, 1.28, 1.29)
- **Testing Period**: 1 month testing before general availability
- **Upgrade Process**: 3-week phased rollout
  1. Week 1: User notification
  2. Week 2: Beta testing (optional)
  3. Week 3: Gradual rollout

<Warning>
**Important**: Never manually upgrade Kubernetes from your cloud provider console.
Let Qovery manage the upgrade process to ensure compatibility.
</Warning>

## Cost Optimization

### Cost Factors

1. **Compute**: Node instance types and sizes
2. **Storage**: Persistent volumes and snapshots
3. **Network**: Data transfer and load balancers
4. **Control Plane**: Kubernetes master nodes (varies by provider)
5. **Add-ons**: Managed databases, object storage, etc.

### Optimization Strategies

<Tabs>
  <Tab title="Right-sizing">
    - Start small, scale as needed
    - Use auto-scaling
    - Monitor resource usage
    - Adjust based on metrics
    - Use burst-able instances for dev
  </Tab>

  <Tab title="Reserved Instances">
    - 1-year or 3-year commitments
    - 30-70% savings
    - Best for production workloads
    - Available on AWS, GCP, Azure
  </Tab>

  <Tab title="Spot Instances">
    - 50-90% savings
    - For fault-tolerant workloads
    - Qovery supports spot node pools
    - Automatic fallback to on-demand
  </Tab>

  <Tab title="Environment Management">
    - Stop dev/staging when not in use
    - Use preview environments (auto-cleanup)
    - Delete unused resources
    - Monitor costs with Qovery
  </Tab>
</Tabs>
