## Quick Start

### Connect a Git Repository

<Steps>
  <Step title="Navigate to Organization Settings">
    Go to your organization settings in the Qovery Console
  </Step>

  <Step title="Select Git Provider">
    Click on **Git Providers** and choose your provider (GitHub, GitLab, or Bitbucket)
  </Step>

  <Step title="Authorize Access">
    Follow the OAuth flow to authorize Qovery to access your repositories
  </Step>

  <Step title="Start Deploying">
    Your repositories are now available when creating applications
  </Step>
</Steps>

### Connect a Container Registry

<Steps>
  <Step title="Go to Organization Settings">
    Navigate to **Settings** → **Container Registries**
  </Step>

  <Step title="Add Registry">
    Click **Add Registry** and select your registry type
  </Step>

  <Step title="Provide Credentials">
    Enter registry URL and authentication credentials
  </Step>

  <Step title="Deploy Images">
    Deploy container images from your connected registry
  </Step>
</Steps>

## Popular Integrations

### AWS

Complete AWS integration for enterprise deployments:

- **EKS**: Managed Kubernetes clusters
- **ECR**: Container image registry
- **RDS**: Managed databases (PostgreSQL, MySQL)
- **S3**: Object storage for backups and artifacts
- **Secrets Manager**: Centralized secret management
- **CloudWatch**: Logging and monitoring

[Learn more →](/integrations/cloud-providers/aws)

### GitHub

Full GitHub integration for GitOps workflows:

- **GitHub Repositories**: Deploy from any repository
- **GitHub Actions**: CI/CD integration with Qovery CLI
- **GitHub Container Registry**: Store and deploy container images
- **Preview Environments**: Automatic environments for pull requests

[Learn more →](/integrations/git-providers/github)

### Docker Hub

Use Docker Hub for container images:

- **Public Images**: Deploy from Docker Hub's public registry
- **Private Repositories**: Connect your Docker Hub account
- **Official Images**: Use verified images for databases and services

[Learn more →](/integrations/container-registries/docker-hub)

## Multi-Cloud Support

Qovery supports multiple cloud providers, allowing you to:

- **Avoid Vendor Lock-in**: Switch providers without changing your workflow
- **Multi-Cloud Deployments**: Deploy different environments on different clouds
- **Hybrid Cloud**: Combine cloud and on-premise Kubernetes clusters
- **Disaster Recovery**: Replicate across multiple cloud providers

### Supported Cloud Providers

| Provider | Kubernetes Service | Container Registry | Managed Databases | Object Storage |
|----------|-------------------|-------------------|-------------------|----------------|
| **AWS** | EKS | ECR | RDS | S3 |
| **Google Cloud** | GKE | Artifact Registry | Cloud SQL | Cloud Storage |
| **Azure** | AKS | ACR | Azure Database | Blob Storage |
| **Scaleway** | Kapsule | Container Registry | Managed Databases | Object Storage |
| **Self-Managed** | Any Kubernetes | Any Registry | Container Mode | Any S3-compatible |

## Infrastructure as Code

Qovery integrates with Infrastructure as Code tools:

### Terraform Provider

Manage Qovery resources with Terraform:

```hcl
resource "qovery_application" "my_app" {
  name = "my-application"
  environment_id = qovery_environment.production.id

  git_repository = {
    url = "https://github.com/my-org/my-app"
    branch = "main"
  }

  build_mode = "DOCKER"
  dockerfile_path = "Dockerfile"
}
```

[View Terraform docs →](/terraform/overview)

### Qovery CLI

Automate deployments with the CLI:

```bash
# Deploy application
qovery application deploy --application-name my-app

# Create environment
qovery environment create --name staging

# Clone environment
qovery environment clone --from production --to staging-test
```

[View CLI docs →](/cli/overview)
