## Quick Start Guides

### 1. Connect Your Infrastructure

<Steps>
  <Step title="Choose Kubernetes Platform">
    Select your Kubernetes deployment option:
    - **Qovery-Managed**: Automated cluster on AWS, GCP, Azure, or Scaleway
    - **BYOK**: Connect your existing Kubernetes cluster
    - **Local**: Docker Desktop for development

    [Kubernetes Setup →](/integrations/kubernetes/overview)
  </Step>

  <Step title="Connect Git Repository">
    1. Go to Organization Settings → Git Providers
    2. Select GitHub, GitLab, or Bitbucket
    3. Authorize Qovery to access repositories
    4. Your repos are now available for deployments

    [Git Providers Guide →](/integrations/git-providers/overview)
  </Step>

  <Step title="Optional: Add Container Registry">
    Connect external registries:
    - Docker Hub for public images
    - ECR/GCR/ACR for cloud-native workflows
    - Private registries for internal images

    [Registry Setup →](/integrations/container-registries/overview)
  </Step>
</Steps>

### 2. Set Up CI/CD

<Steps>
  <Step title="Choose CI/CD Approach">
    **Option A: Qovery Deploy** (Zero config)
    - Built-in build optimization
    - Auto-deploy on Git push
    - Build insights (coming soon)

    **Option B: External CI/CD** (Full control)
    - GitHub Actions, GitLab CI, CircleCI, Jenkins
    - Build in CI, deploy with Qovery
    - Run tests and security scans
  </Step>

  <Step title="Configure Deployment">
    - Enable auto-deploy for automatic deployments
    - Or use Qovery CLI in your CI/CD pipeline
    - Set up preview environments for PRs

    [CI/CD Integration →](/integrations/ci-cd/overview)
  </Step>
</Steps>

### 3. Add Observability

<Steps>
  <Step title="Start with Qovery Observe">
    Built-in observability with zero configuration:
    - Real-time log streaming
    - CPU, memory, network metrics
    - Kubernetes events
    - 90-day retention (with package)
  </Step>

  <Step title="Optional: Add External Tools">
    Deploy via Helm for advanced features:
    - Datadog for full-stack APM
    - New Relic for application monitoring
    - Prometheus + Grafana for custom metrics
    - Elastic for log analysis

    [Observability Options →](/integrations/observability/overview)
  </Step>
</Steps>

### Terraform Module Deployment

Deploy and manage Terraform modules directly from Git:

```hcl
# Deploy AWS infrastructure with Terraform on Qovery
resource "qovery_helm" "terraform_module" {
  environment_id = qovery_environment.prod.id
  name           = "infrastructure"

  source = {
    git = {
      url    = "https://github.com/my-org/terraform-modules"
      branch = "main"
      path   = "aws/vpc"
    }
  }

  # Qovery manages state automatically
  # Variables injected securely
  # Outputs available as environment variables
}
```

**Features**:
- Automatic state management
- Secure variable injection
- Plan & apply workflow
- Drift detection
- Terragrunt support (coming soon)

## Multi-Cloud Strategy

### Avoid Vendor Lock-in

Deploy across multiple cloud providers without changing your workflow:

| Use Case | Recommended Setup |
|----------|------------------|
| **Cost Optimization** | Dev on Scaleway, Prod on AWS |
| **Geographic Distribution** | US on AWS, EU on GCP, Asia on Azure |
| **Disaster Recovery** | Primary on AWS, Backup on GCP |
| **Compliance** | Data residency per region/country |
| **Development** | Local Docker Desktop, Cloud staging/prod |

### Provider Comparison

| Provider | Best For | Kubernetes | Pricing | Regions |
|----------|----------|------------|---------|---------|
| **AWS EKS** | Enterprise, most services | Excellent | $$$ | Most regions globally |
| **GCP GKE** | Data/ML workloads | Excellent | $$ | Good global coverage |
| **Azure AKS** | Microsoft ecosystem | Excellent | $$ | Good global coverage |
| **Scaleway** | Cost-conscious, EU data | Good | $ | Europe only |
| **BYOK** | Maximum control | Depends | Your cost | Anywhere |

## Infrastructure as Code

### Terraform Provider

Manage your entire Qovery infrastructure as code:

```hcl
# Complete application stack
resource "qovery_environment" "prod" {
  project_id = var.project_id
  cluster_id = var.cluster_id
  name       = "production"
  mode       = "PRODUCTION"
}

resource "qovery_database" "postgres" {
  environment_id = qovery_environment.prod.id
  name           = "main-db"
  type           = "POSTGRESQL"
  version        = "15"
  mode           = "MANAGED"
}

resource "qovery_application" "api" {
  environment_id = qovery_environment.prod.id
  name           = "api"

  git_repository = {
    url    = "https://github.com/my-org/api"
    branch = "main"
  }

  # Database connection automatically injected
  depends_on = [qovery_database.postgres]
}
```

[View Terraform examples →](/terraform/examples)

### Qovery CLI

Automate deployments and manage resources programmatically:

```bash
# Deploy application
qovery application deploy --application-name api --watch

# Create and deploy environment
qovery environment create --name staging --clone-from production

# Port forward for debugging
qovery port-forward --application api --port 8080:3000

# Execute commands in container
qovery shell --application api
```

[CLI Documentation →](/cli/overview)
