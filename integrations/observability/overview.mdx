## Overview

Qovery provides comprehensive observability through **Qovery Observe** (built-in) and integrations with leading monitoring platforms. Deploy external observability tools using their official Helm charts directly within your Qovery environments.

## Qovery Observe vs External Tools

| Feature | Qovery Observe | External Tools (Helm) |
|---------|----------------|----------------------|
| **Setup** | Built-in, no configuration | Deploy as Helm service |
| **Cost** | Included with Qovery | Tool-specific pricing |
| **Log Retention** | 90 days (with package) | Based on tool configuration |
| **Metrics** | CPU, memory, network | Full custom metrics |
| **APM** | Basic | Advanced (Datadog) |
| **Alerts** | Via Qovery Console | Native tool alerting |
| **Integration** | Deep Qovery integration | Standard Kubernetes |

## Quick Start

### Using Qovery Observe

Qovery Observe is enabled by default for all applications:

<Steps>
  <Step title="Deploy Application">
    Deploy any application to your Qovery environment
  </Step>

  <Step title="View Logs">
    Navigate to application â†’ Logs tab for real-time log streaming
  </Step>

  <Step title="Check Metrics">
    View CPU, memory, and network metrics in the Metrics tab
  </Step>

  <Step title="Monitor Events">
    Track Kubernetes and deployment events in the Events tab
  </Step>
</Steps>

### Deploying External Tools

Deploy observability tools as Helm services:

<Steps>
  <Step title="Add Helm Service">
    Create â†’ Helm Chart in your environment
  </Step>

  <Step title="Select Official Chart">
    Choose from:
    - Datadog: `datadog/datadog`
    - Prometheus: `prometheus-community/kube-prometheus-stack`
    - Grafana: `grafana/grafana`
  </Step>

  <Step title="Configure Values">
    Set API keys, endpoints, and configurations
  </Step>

  <Step title="Deploy">
    Deploy the Helm service to start monitoring
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Logs Not Showing">
    **Problem**: No logs visible in Qovery Observe

    **Solutions**:
    - Verify application is running and producing logs
    - Check application writes to stdout/stderr
    - Ensure Observability Package is enabled (for extended retention)
    - Wait a few seconds for log aggregation
    - Check pod status and events
  </Accordion>

  <Accordion title="External Tool Not Collecting Data">
    **Problem**: Helm-deployed tool not receiving metrics/logs

    **Solutions**:
    - Verify Helm deployment is running (check pods)
    - Confirm API keys/credentials are correct
    - Check network policies allow traffic
    - Review tool-specific configuration
    - Check tool's agent logs for errors
  </Accordion>

  <Accordion title="High Observability Costs">
    **Problem**: External tool costs increasing

    **Solutions**:
    - Review data ingestion rates
    - Optimize log sampling (not all logs need collection)
    - Use Qovery Observe for non-production
    - Set up data retention policies
    - Filter noisy logs at source
  </Accordion>

  <Accordion title="Missing Metrics">
    **Problem**: Some metrics not appearing

    **Solutions**:
    - Verify application exposes metrics endpoint
    - Check ServiceMonitor/PodMonitor configuration
    - Ensure metrics port is accessible
    - Review Prometheus scraping config
    - Check application instrumentation
  </Accordion>
</AccordionGroup>
